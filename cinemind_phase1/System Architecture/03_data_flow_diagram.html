CineMind: Data Flow Diagram

How a piece of data actually moves through the system: offline batch processing builds the indexes once, an online request path serves each user in real time, and a feedback loop closes the cycle back into retraining.

LANE A Offline Batch Flow

Runs once (and again on each retrain) — never on the request path. Produces the vectors that Lane B searches against.

1 MovieLens 100K Raw ratings & item metadata pulled in
2 OMDb title+year match No links.csv in ml-100k — matched by title/year instead of joined; poster/plot/cast
3 PostgreSQL Canonical metadata store
4a E5-large-v2 Content embedding: title+plot+genres+cast+keywords
4b Two-Tower training Implicit feedback (rating≥4), in-batch negatives + logQ correction, temporal split
5 Qdrant Content vectors + collaborative vectors indexed for ANN
Writes: artifacts/content_vecs.npy · item_vecs.npy · two_tower.pt → loaded into Qdrant collections
LANE B Online Request Flow

Triggered per user request against the FastAPI backend. Reads from what Lane A built; every LLM hop is traced in Langfuse.

1 Client request React (or Streamlit) → GET /recommendations or POST /chat
2 Redis cache check Cache hit → skip to step 6
3 Candidate generation Cosine search over content vectors → ~30 items
4 Ranking Two-tower dot product + 0.30 × log-popularity prior
5 LLM re-rank + explain LangChain + Claude/Groq: top ~30 → final 10, grounded "Why this?" per item
6 Response Cached in Redis (5 min TTL) → returned to client as recommendation cards
Cold-start branch: no user history → POST /onboarding chat replaces step 3–4 with content-vector search on stated preferences. Independent branch: GET /graph/insights/{movie_id} queries Neo4j directly (co-raters, shared-genre movies) — not part of this numbered pipeline, triggered on demand by a "Graph insights" button per card.
FEEDBACK LOOP — closes back into Lane A
LANE C Feedback & Retraining Loop

The only path that writes new implicit-feedback signal back into the system, closing the loop into Lane A's next retrain.

1 Thumbs up/down, clicks Captured in React/Streamlit UI
2 POST /feedback FastAPI route → feedback.py
3 PostgreSQL feedback log Appended alongside historical ratings
4 Next retrain cycle Re-enters Lane A step 1 as new implicit feedback
Every LLM call across all three lanes is traced by Langfuse: cost, latency, prompt version, and — where available — the feedback signal it led to
