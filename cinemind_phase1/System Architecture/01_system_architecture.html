CineMind: Complete System Architecture

Six-layer architecture for the LLM-augmented hybrid deep learning recommender — from raw data ingestion through to the React client. Each band below is one architectural layer; arrows show the primary direction of data and control flow.

L1 · Data
L2 · Embedding Pipeline
L3 · Storage
L4 · Online Pipeline
L5 · Backend (FastAPI)
L6 · Frontend (React)
L1 Data
Raw sources, joined and normalised into relational storage
MovieLens 100KUser ratings, timestamps — source of implicit feedback (rating ≥ 4)
OMDb APIPlot, cast, posters — matched by title+year (ml-100k ships no links.csv/tmdbId), multi-key rotation for quota
PostgreSQLClean, joined metadata land here before downstream processing
L2 Embedding Pipeline
Offline, batch — runs once (or on retrain), not per-request
Content embeddingsintfloat/E5-large-v2 (local) over title + plot + genres + cast + keywords
Collaborative embeddingsTwo-tower PyTorch model trained on implicit feedback; 64-dim towers
torchsentence-transformersin-batch negativeslogQ correctiontemporal split
L3 Storage
Four purpose-built stores, each serving a different access pattern
QdrantVector search — collaborative vectors (content vectors searched via plain cosine in recommender.py)
Neo4jGraph: user → rated → movie → has_genre → genre. Queried for co-rater & shared-genre insights (GET /graph/insights)
RedisCache-aside for GET /recommendations, 5 min TTL
PostgreSQLFeedback logs (SQLite fallback)
L4 Online Pipeline
Executes per request — Langfuse traces every LLM call in this layer
Candidate generation Cosine search over content vectors → ~30 candidates
Ranking Two-tower dot product + 0.30 × log-popularity prior, Redis-cached 5 min
LLM reasoning (LangChain) (a) conversational query parsing — RAG · (b) re-rank top ~30 → final 10 · (c) grounded "Why this?" explanation · (d) cold-start onboarding chat
Graph insights Neo4j traversal: co-raters & shared-genre movies, independent of the ranking/LLM steps above
LangChain LCELClaude Haiku / Sonnet or GroqLangfuse traces
L5 Backend
FastAPI — orchestrates L1–L4, exposed via 7 routes, containerised alongside its stores
GET /recommendations/{user_id}Full pipeline: candidate gen → rank (Redis-cached) → LLM re-rank
POST /chatConversational search (RAG over content vectors)
POST /onboardingCold-start dialogue for new users
POST /feedbackClicks / ratings → Postgres → retraining
GET /explain/{movie_id}Detailed, metadata-grounded explanation
GET /graph/insights/{movie_id}Co-raters & shared-genre movies via Neo4j
GET /healthLLM provider, feedback backend, cache, graph status
docker-compose: API + Qdrant + Postgres + Redis + Neo4j
L6 Frontend
React — consumes the FastAPI routes above; a parallel Streamlit demo bypasses this layer for public access
Recommendation cardsPoster + score + expandable "Why this?" panel
Graph insights panelCo-raters & shared-genre movies via GET /graph/insights
Chat interfaceLLM dialogue for conversational search
Onboarding wizardCold-start flow for new users
Thumbs up / downFeedback capture → POST /feedback
