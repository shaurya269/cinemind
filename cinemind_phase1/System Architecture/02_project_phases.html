Phase 1 Jupyter Prototype
notebooks/ · src/ · artifacts/

Research and model-development phase. Reproduces the reference repo's RBM baseline, then trains the two-tower model and beats it. Each notebook has a parallel src/*.py script and, where it has run to completion, writes artifacts into artifacts/.

NotebookPurposeProducesStatus
01_data_exploration.ipynb Load MovieLens + TMDB, EDA, join via links.csv artifacts/eda_overview.png done
02_rbm_baseline.ipynb Reproduce reference-repo RBM, measure Precision@10 artifacts/results.csv (baseline row) done
03_content_embeddings.ipynb E5-large-v2 embeddings → test "movies like X" search artifacts/content_vecs.npy, content_movie_ids.npy done
04_two_tower.ipynb Train two-tower model, beat RBM baseline artifacts/two_tower.pt, item_vecs.npy done
05_qdrant_indexing.ipynb Load item_vecs.npy into Qdrant, test ANN search Qdrant collection 'collab_vectors' (live, via Docker) done
06_llm_layer.ipynb LangChain chains: parse → retrieve → rerank → explain, traced in Langfuse Executed end-to-end via Groq (free tier) — Anthropic optional done
07_evaluation.ipynb Compare popularity / RBM / two-tower / hybrid side by side artifacts/results.csv (full comparison) done
src/07_omdb_enrichment.py (no notebook mirror — batch script) Poster/plot/cast via OMDb, title+year matched (multi-key rotation, resumable) artifacts/movie_meta.csv (1,548/1,682 movies matched) done

Phase 3.5 Streamlit Public Demo
streamlit_app/

Parallel, lightweight deployment track — not a replacement for Phases 2–3. Calls recommender.py and llm_chains.py directly (no API hop) so recruiters/reviewers can try the system without Docker infra. Degrades gracefully: Qdrant → in-memory fallback, Postgres → SQLite, Neo4j/Redis skipped.

FilePurposeStatus
